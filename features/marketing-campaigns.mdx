## Key Capabilities

### Promoted Listings Campaigns

**Drive Traffic with Pay-Per-Sale Advertising**

Promoted Listings increase your product visibility in eBay search results. You only pay when an item sells:

```typescript
// Ask your AI assistant:
"Create a promoted listings campaign for my electronics category with an 8% ad rate"

// The AI will execute:
const campaign = await mcp.useTool('marketing_createCampaign', {
  campaign_name: 'Electronics - Summer Sale 2024',
  marketplace_id: 'EBAY_US',
  funding_strategy: 'COST_PER_SALE',
  campaign_criterion: {
    criterion_type: 'INVENTORY_PARTITION',
    selection_rules: [
      {
        categories: [
          { category_id: '293', category_scope: 'ALL_INCLUDING_SUBCATEGORIES' }
        ]
      }
    ]
  }
});

// Set ad rate (percentage of sale price)
await mcp.useTool('marketing_updateAdRate', {
  campaign_id: campaign.campaignId,
  ad_rate: 8 // 8% of sale price
});

console.log(`‚úÖ Campaign created: ${campaign.campaignId}`);
```

<Note>
**Ad Rate Guidelines:**
- **Competitive categories:** 8-12% for electronics, fashion
- **Less competitive:** 3-5% for unique items
- **High margin:** Can afford 10-15% for fast sales
- eBay recommends starting at 5% and adjusting based on performance
</Note>

### Campaign Performance Tracking

**Measure ROI and Optimize Campaigns**

Track impressions, clicks, conversions, and sales from your campaigns:

```typescript
"Show me performance for my Electronics campaign"

const report = await mcp.useTool('marketing_getCampaignReport', {
  campaign_id: 'campaign-123',
  metric_keys: ['CLICK', 'IMPRESSION', 'SALE', 'AD_REVENUE']
});

// Analyze results
const metrics = {
  impressions: report.metrics.impression,
  clicks: report.metrics.click,
  sales: report.metrics.sale,
  revenue: report.metrics.adRevenue.value,
  ctr: (report.metrics.click / report.metrics.impression * 100).toFixed(2),
  conversionRate: (report.metrics.sale / report.metrics.click * 100).toFixed(2)
};

console.log(`
üìä Campaign Performance:
   Impressions: ${metrics.impressions.toLocaleString()}
   Clicks: ${metrics.clicks.toLocaleString()}
   Sales: ${metrics.sales}
   Ad Revenue: $${metrics.revenue}
   CTR: ${metrics.ctr}%
   Conversion Rate: ${metrics.conversionRate}%
`);
```

**Available Metrics:**
- `IMPRESSION` - Times ad was shown
- `CLICK` - Times buyers clicked your listing
- `SALE` - Number of sales from promoted listings
- `AD_REVENUE` - Total sales revenue from promoted items
- `AD_SPEND` - Total ad fees paid
- `ROAS` - Return on ad spend (calculated)

## Real-World Use Cases

### Use Case 1: Seasonal Campaign Automation

**Problem:** Need to run holiday promotions but don't want to manually manage timing and discounts.

**Solution:** AI-scheduled seasonal campaigns with automatic start/stop.

```typescript
"Create a Black Friday campaign: 30% off all items, promoted at 10% ad rate,
running from November 24-27"

async function createBlackFridayCampaign() {
  // 1. Create markdown sale
  const promotion = await mcp.useTool('marketing_createItemPromotion', {
    marketplace_id: 'EBAY_US',
    name: 'Black Friday Mega Sale - 30% Off Everything',
    promotion_type: 'MARKDOWN_SALE',
    start_date: '2024-11-24T00:00:00Z',
    end_date: '2024-11-27T23:59:59Z',
    discount_rules: [
      {
        discount_type: 'PERCENTAGE',
        discount_amount: '30',
        apply_to_all_inventory: true
      }
    ]
  });

  // 2. Create promoted listings campaign
  const campaign = await mcp.useTool('marketing_createCampaign', {
    campaign_name: 'Black Friday Promoted Listings',
    marketplace_id: 'EBAY_US',
    funding_strategy: 'COST_PER_SALE',
    campaign_criterion: {
      criterion_type: 'INVENTORY_PARTITION',
      selection_rules: [{ apply_to_all_inventory: true }]
    }
  });

  await mcp.useTool('marketing_updateAdRate', {
    campaign_id: campaign.campaignId,
    ad_rate: 10
  });

  console.log(`‚úÖ Black Friday campaign ready!`);
  console.log(`   Promotion: ${promotion.promotionId}`);
  console.log(`   Campaign: ${campaign.campaignId}`);

  return { promotion, campaign };
}
```

<Tip>
Schedule this automation to run a week before Black Friday to ensure campaigns are approved and live on time.
</Tip>

### Use Case 2: Competitor-Based Dynamic Pricing

**Problem:** Competitors constantly undercut your prices, making manual repricing impossible.

**Solution:** AI-monitored competitive pricing with automatic promotion adjustments.

```typescript
"Monitor competitor prices and adjust my promotions to stay competitive"

async function dynamicCompetitivePricing() {
  // Get current active promotions
  const promotions = await mcp.useTool('marketing_getPromotions', {
    marketplace_id: 'EBAY_US',
    promotion_status: 'RUNNING'
  });

  for (const promotion of promotions.promotions) {
    // Analyze market pricing (using eBay analytics or external data)
    const marketData = await analyzeMarketPrices(promotion);

    // If competitors are offering better deals, increase discount
    if (marketData.avgCompetitorDiscount > promotion.discountAmount) {
      // End current promotion
      await mcp.useTool('marketing_endPromotion', {
        promotion_id: promotion.promotionId
      });

      // Create new one with better discount
      const newDiscount = marketData.avgCompetitorDiscount + 5; // Beat by 5%

      await mcp.useTool('marketing_createItemPromotion', {
        marketplace_id: 'EBAY_US',
        name: `Competitive Pricing - ${newDiscount}% Off`,
        promotion_type: 'MARKDOWN_SALE',
        start_date: new Date().toISOString(),
        end_date: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000).toISOString(),
        discount_rules: [
          {
            discount_type: 'PERCENTAGE',
            discount_amount: newDiscount.toString(),
            inventory_items: promotion.inventoryItems
          }
        ]
      });

      console.log(`üîÑ Updated promotion to ${newDiscount}% off`);
    }
  }
}

// Run every 6 hours
setInterval(dynamicCompetitivePricing, 6 * 60 * 60 * 1000);
```

### Use Case 3: Performance-Based Campaign Optimization

**Problem:** Some products perform well with ads, others don't‚Äîwasting ad spend.

**Solution:** Automated campaign optimization based on ROI.

```typescript
"Analyze my promoted listings and pause campaigns with ROI below 3x"

async function optimizeCampaignsByROI() {
  const campaigns = await mcp.useTool('marketing_getCampaigns', {
    marketplace_id: 'EBAY_US',
    campaign_status: 'RUNNING'
  });

  for (const campaign of campaigns.campaigns) {
    // Get performance report
    const report = await mcp.useTool('marketing_getCampaignReport', {
      campaign_id: campaign.campaignId,
      metric_keys: ['AD_REVENUE', 'AD_SPEND', 'SALE', 'IMPRESSION', 'CLICK']
    });

    // Calculate ROI
    const adSpend = parseFloat(report.metrics.adSpend?.value || 0);
    const adRevenue = parseFloat(report.metrics.adRevenue?.value || 0);
    const roi = adSpend > 0 ? adRevenue / adSpend : 0;

    console.log(`Campaign: ${campaign.campaignName}`);
    console.log(`  Spend: $${adSpend.toFixed(2)}`);
    console.log(`  Revenue: $${adRevenue.toFixed(2)}`);
    console.log(`  ROI: ${roi.toFixed(2)}x`);

    // Pause low-performing campaigns
    if (roi < 3.0 && adSpend > 50) { // Min $50 spend for statistical significance
      await mcp.useTool('marketing_pauseCampaign', {
        campaign_id: campaign.campaignId
      });
      console.log(`  ‚è∏Ô∏è  Paused (low ROI)`);
    }

    // Increase ad rate for high performers
    else if (roi > 5.0) {
      const currentRate = campaign.adRate || 5;
      const newRate = Math.min(currentRate + 2, 15); // Cap at 15%

      await mcp.useTool('marketing_updateAdRate', {
        campaign_id: campaign.campaignId,
        ad_rate: newRate
      });
      console.log(`  üìà Increased ad rate to ${newRate}%`);
    }
  }
}
```

## Best Practices

  <Accordion title="Ad Rate Optimization" icon="percentage">
    **Start low, scale based on performance:**

    1. **Initial launch:** 3-5% ad rate
    2. **Monitor for 7 days:** Track impressions and sales
    3. **Adjust based on results:**
       - Low impressions? Increase by 2%
       - Good ROI (>4x)? Increase by 1-2%
       - Poor ROI (<2x)? Decrease or pause

    ```typescript
      { adRate: 8, roi: 4.1 },
      { adRate: 12, roi: 3.8 }
    ];

    // 8% provides best ROI
    const optimalRate = 8;
    ```

    **Category-specific rates:**
    - Electronics: 7-10%
    - Fashion: 10-12%
    - Collectibles: 5-8%
    - Automotive: 4-6%
  </Accordion>

  <Accordion title="Promotion Strategy" icon="calendar">

    - **January:** New Year clearance (20-30% off)
    - **February:** Valentine's Day (gifts category)
    - **May:** Spring cleaning sales
    - **July:** Summer clearance

    **Discount levels:**
    - 10-15%: Minor promotions

## Troubleshooting

<AccordionGroup>
  <Accordion title="Campaign not showing in search">
    **Cause:** Ad rate too low or insufficient inventory.

    **Solution:**
    - Increase ad rate by 2-3%
    - Ensure items are active and available
    - Check that items meet eBay's promoted listings eligibility
    - Wait 24 hours for campaign approval
  </Accordion>

  <Accordion title="Promotion not applying at checkout">
    **Cause:** Buyer doesn't meet promotion requirements or promotion expired.

    **Solution:**
    - Verify promotion dates (start/end)
    - Check minimum order value or quantity requirements
    - Ensure items are included in promotion
    - Promotions can take 1-2 hours to go live
  </Accordion>

  <Accordion title="High ad spend, low sales">
    **Cause:** Targeting wrong products or ad rate too high.

    **Solution:**
    - Pause underperforming campaigns
    - Analyze which products convert best
    - Reduce ad rate and test
    - Focus on high-margin products
    - Improve listing quality (images, descriptions)
  </Accordion>

  <Accordion title="Cannot create promotion">
    **Cause:** Conflicting promotions or invalid parameters.

    **Solution:**
    - Check for existing promotions on same items
    - Verify date ranges don't overlap
    - Ensure discount amount is within limits (0-75%)
    - Confirm promotion type is supported in your marketplace
  </Accordion>
</AccordionGroup>
