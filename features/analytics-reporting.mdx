Track how buyers discover and interact with your listings:

**Revenue and Profit Analysis**

Track sales performance and identify trends:

```typescript
"Analyze my sales performance for Q1 2024"

async function quarterlySalesAnalysis() {
  const report = await mcp.useTool('analytics_getTrafficReport', {
    dimension: 'DAY',
    metric_keys: ['TRANSACTION', 'TRANSACTION_AMOUNT'],
    start_date: '2024-01-01',
    end_date: '2024-03-31',
    sort: 'day'
  });

  // Calculate totals
  let totalSales = 0;
  let totalRevenue = 0;

  report.records.forEach(day => {
    totalSales += day.transaction || 0;
    totalRevenue += parseFloat(day.transaction_amount?.value || 0);

  const dailyAvgSales = totalSales / 90; // 90 days in Q1

  console.log(`
ğŸ’° Q1 2024 Sales Report:

   Best Day: ${findBestDay(report.records)}

  `);

  return { totalSales, totalRevenue, avgOrderValue };
}
```

## Real-World Use Cases

### Use Case 1: Listing Optimization

**Problem:** Some listings get lots of views but don't convert to sales.

**Solution:** AI-powered listing analysis and optimization recommendations.

```typescript
"Analyze my listings and identify those with high views but low conversion"

async function identifyUnderperformers() {
  const report = await mcp.useTool('analytics_getTrafficReport', {
    dimension: 'LISTING',
    metric_keys: ['LISTING_VIEWS', 'SALES_CONVERSION_RATE', 'TRANSACTION'],
    start_date: '2024-01-01',
    end_date: '2024-01-31',
    filter: 'listing_views:>100' // Only listings with 100+ views
  });

  // Find high-traffic, low-conversion listings
  const underperformers = report.records
    .filter(listing => listing.sales_conversion_rate < 2.0) // < 2% conversion
    .sort((a, b) => b.listing_views - a.listing_views)
    .slice(0, 10);

  console.log('âš ï¸  Underperforming Listings (High Views, Low Sales):');

  for (const listing of underperformers) {
    console.log(`
ğŸ“‰ ${listing.title}
   Views: ${listing.listing_views}
   Sales: ${listing.transaction}
   Conversion: ${listing.sales_conversion_rate}%

   Recommendations:
   - Check pricing vs. competitors
   - Improve product description
   - Add more/better images
   - Offer free shipping or promotions
   - Review item specifics accuracy
    `);
  }

  return underperformers;
}
```

<Tip>
Target conversion rate benchmarks by category:
- Electronics: 2-3%
- Fashion: 1.5-2.5%
- Collectibles: 1-2%
- Home & Garden: 2-4%
</Tip>

### Use Case 2: Inventory Planning

**Problem:** Need to know which products to restock based on sales velocity.

**Solution:** Data-driven inventory forecasting.

```typescript
"Analyze product performance and recommend restocking priorities"

async function inventoryForecast() {
  // Get sales data
  const salesReport = await mcp.useTool('analytics_getTrafficReport', {
    dimension: 'LISTING',
    metric_keys: ['TRANSACTION', 'LISTING_VIEWS', 'SALES_CONVERSION_RATE'],
    start_date: '2024-01-01',
    end_date: '2024-01-31'
  });

  // Get current inventory
  const inventory = await mcp.useTool('inventory_getInventoryItems', {
    limit: 200
  });

  // Match sales to inventory
  const restockPriorities = [];

  for (const item of inventory.inventoryItems) {
    const salesData = salesReport.records.find(r => r.sku === item.sku);

    if (salesData) {
      const monthlySales = salesData.transaction;
      const currentQty = item.availability.quantity;
      const daysOfStock = (currentQty / monthlySales) * 30;

      restockPriorities.push({
        sku: item.sku,
        title: item.product.title,
        currentQty,
        monthlySales,
        daysOfStock,
        priority: daysOfStock < 15 ? 'HIGH' : daysOfStock < 30 ? 'MEDIUM' : 'LOW'
      });
    }
  }

  // Sort by urgency
  restockPriorities
    .sort((a, b) => a.daysOfStock - b.daysOfStock)
    .forEach(item => {
      if (item.priority === 'HIGH') {
        console.log(`
ğŸ”´ ${item.title}
   Current Stock: ${item.currentQty}
   Monthly Sales: ${item.monthlySales}
   Days Remaining: ${item.daysOfStock.toFixed(0)}
   Action: RESTOCK NOW
        `);
      }
    });

  return restockPriorities;
}
```

### Use Case 3: Marketing ROI Dashboard

**Problem:** Need to correlate marketing campaigns with sales and traffic increases.

**Solution:** Unified analytics dashboard combining traffic, sales, and marketing data.

```typescript
"Create a comprehensive marketing ROI report"

async function marketingROIDashboard() {
  // Get traffic data
  const traffic = await mcp.useTool('analytics_getTrafficReport', {
    dimension: 'DAY',
    metric_keys: ['LISTING_VIEWS', 'TRANSACTION', 'TRANSACTION_AMOUNT'],
    start_date: '2024-01-01',
    end_date: '2024-01-31'
  });

  // Get marketing campaign performance
  const campaigns = await mcp.useTool('marketing_getCampaigns', {
    marketplace_id: 'EBAY_US'
  });

  let totalAdSpend = 0;
  let totalAdRevenue = 0;

  for (const campaign of campaigns.campaigns) {
    const report = await mcp.useTool('marketing_getCampaignReport', {
      campaign_id: campaign.campaignId,
      metric_keys: ['AD_SPEND', 'AD_REVENUE', 'IMPRESSION', 'CLICK', 'SALE']
    });

    totalAdSpend += parseFloat(report.metrics.adSpend?.value || 0);
    totalAdRevenue += parseFloat(report.metrics.adRevenue?.value || 0);
  }

  // Calculate overall revenue
  const organicRevenue = traffic.records.reduce((sum, day) => {
    return sum + parseFloat(day.transaction_amount?.value || 0);
  }, 0);

  const totalRevenue = organicRevenue;
  const adROI = totalAdRevenue / totalAdSpend;

  console.log(`
ğŸ“Š Marketing ROI Dashboard - January 2024

ğŸ’° Revenue:
   Total Revenue: $${totalRevenue.toLocaleString()}
   Organic Revenue: $${organicRevenue.toLocaleString()}
   Ad Revenue: $${totalAdRevenue.toLocaleString()}

ğŸ“¢ Advertising:
   Total Ad Spend: $${totalAdSpend.toLocaleString()}
   ROAS: ${adROI.toFixed(2)}x
   Ad Revenue %: ${(totalAdRevenue / totalRevenue * 100).toFixed(1)}%

ğŸ“ˆ Efficiency:
   Cost per Sale: $${(totalAdSpend / traffic.records.reduce((s, d) => s + d.transaction, 0)).toFixed(2)}
   Profit Margin: ${((totalRevenue - totalAdSpend) / totalRevenue * 100).toFixed(1)}%
  `);

  return { totalRevenue, totalAdSpend, adROI };
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="No data in analytics reports">
    **Cause:** Insufficient permissions or recent account setup.

    **Solution:**
    - Verify user token has analytics permissions
    - Wait 24-48 hours for new accounts to populate data
    - Check date range isn't in the future
    - Ensure you have active listings during the period
  </Accordion>

  <Accordion title="Metrics don't match Seller Hub">
    **Cause:** Different calculation methods or timezone differences.

    **Solution:**
    - Analytics API uses UTC timezone
    - Seller Hub may use different attribution windows
    - Check that date ranges exactly match
    - Some metrics update with a 24-48 hour delay
  </Accordion>

  <Accordion title="Traffic report returns empty">
    **Cause:** No traffic during the specified period or invalid filters.

    **Solution:**
    - Verify listings were active during date range
    - Check filter syntax (e.g., `listing_views:>100`)
    - Ensure dimension and metrics are compatible
    - Try broader date range
  </Accordion>
</AccordionGroup>
