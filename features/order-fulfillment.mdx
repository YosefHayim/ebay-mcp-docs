## Key Capabilities

### Order Management

**Retrieve and Filter Orders**

Get orders based on status, date range, or specific criteria:

```typescript
// Ask your AI assistant:
"Show me all unfulfilled orders from the last 24 hours"

// The AI will execute:
const orders = await mcp.useTool('fulfillment_getOrders', {
  filter: 'orderfulfillmentstatus:{NOT_STARTED}',
  limit: 50
});

// Process orders
orders.orders.forEach(order => {
  console.log(`Order ${order.orderId}:`);
  console.log(`  Buyer: ${order.buyer.username}`);
  console.log(`  Total: ${order.pricingSummary.total.value} ${order.pricingSummary.total.currency}`);
  console.log(`  Items: ${order.lineItems.length}`);
});
```

<Note>
**Order Statuses:**
- `NOT_STARTED` - Payment received, ready to ship
- `IN_PROGRESS` - Partially fulfilled
- `FULFILLED` - All items shipped
- `CANCELLED` - Order cancelled
</Note>

**Get Specific Order Details**

```typescript
"Get full details for order 12-34567-89012"

const order = await mcp.useTool('fulfillment_getOrder', {
  order_id: '12-34567-89012'
});

// Returns complete order information:
// - Buyer details and shipping address
// - Line items with SKUs and quantities
// - Payment status and pricing
// - Fulfillment instructions
// - Cancellation status (if applicable)
```

### Shipping Label Generation

**Print Shipping Labels**

Generate shipping labels directly from eBay:

```typescript
"Generate a shipping label for order 12-34567-89012 using the buyer's selected service"

const label = await mcp.useTool('fulfillment_createShippingLabel', {
  order_id: '12-34567-89012',
  shipping_service: 'USPS_PRIORITY',
  label_size: 'LABEL_4X6'
});

// Returns:
// - Shipping label PDF URL
// - Tracking number
// - Label cost
// - Label ID for cancellation

console.log(`Label URL: ${label.labelUrl}`);
console.log(`Tracking: ${label.trackingNumber}`);
console.log(`Cost: $${label.cost.value}`);
```

**Supported Carriers:**
- USPS (Priority, First Class, Media Mail, Parcel Select)
- UPS (Ground, 2nd Day Air, Next Day Air)
- FedEx (Ground, Express, 2Day, Standard Overnight)
- DHL Express

### Returns Management

**Process Return Requests**

Handle buyer return requests and issue refunds:

```typescript
"Process return for order 12-34567-89012 and issue full refund"

// 1. Get return request details
const returnRequest = await mcp.useTool('fulfillment_getReturn', {
  return_id: 'return-001'
});

// 2. Approve return
await mcp.useTool('fulfillment_processReturnRequest', {
  return_id: 'return-001',
  decision: 'ACCEPT',
  return_label_provided: true
});

// 3. Issue refund once item received
await mcp.useTool('fulfillment_issueRefund', {
  order_id: '12-34567-89012',
  refund_items: [
    {
      line_item_id: 'item-001',
      refund_amount: {
        value: '29.99',
        currency: 'USD'
      }
    }
  ],
  reason_for_refund: 'BUYER_RETURNED_ITEM'
});
```

<AccordionGroup>
  <Accordion title="Return Reasons">
    **Common return reasons:**
    - `BUYER_RETURNED_ITEM` - Item returned by buyer
    - `ITEM_NOT_AS_DESCRIBED` - Product doesn't match listing
    - `DEFECTIVE_ITEM` - Item is defective or damaged
    - `WRONG_ITEM_SENT` - Incorrect item shipped
    - `BUYER_REMORSE` - Buyer changed mind
  </Accordion>

  <Accordion title="Refund Types">
    **Full vs. Partial Refunds:**

    ```typescript
    // Full refund (item + shipping)
    await mcp.useTool('fulfillment_issueRefund', {
      order_id: '12-34567-89012',
      refund_items: [{ line_item_id: 'item-001' }],
      reason_for_refund: 'DEFECTIVE_ITEM'
    });

    // Partial refund (restocking fee)
    await mcp.useTool('fulfillment_issueRefund', {
      order_id: '12-34567-89012',
      refund_items: [
        {
          line_item_id: 'item-001',
          refund_amount: { value: '24.99', currency: 'USD' } // $5 restocking
        }
      ],
      reason_for_refund: 'BUYER_REMORSE'
    });
    ```
  </Accordion>
</AccordionGroup>

### Order Cancellations

**Cancel Orders**

Handle cancellation requests from buyers or cancel orders yourself:

```typescript
"Cancel order 12-34567-89012 because item is out of stock"

await mcp.useTool('fulfillment_cancelOrder', {
  order_id: '12-34567-89012',
  cancel_reason: 'OUT_OF_STOCK',
  buyer_paid: true // Triggers automatic refund
});
```

**Cancellation Reasons:**
- `OUT_OF_STOCK` - Item no longer available
- `BUYER_ASKED_CANCEL` - Buyer requested cancellation
- `PRICE_ERROR` - Pricing mistake in listing
- `FOUND_CHEAPER_PRICE` - Buyer found better deal
- `ORDER_MISTAKE` - Buyer made a mistake

<Warning>
Frequent order cancellations can negatively impact your seller metrics. Only cancel when absolutely necessary.
</Warning>

## Real-World Use Cases

### Use Case 1: Daily Order Processing

**Problem:** Processing 50+ orders daily is time-consuming and error-prone.

**Solution:** Automated order fulfillment workflow.

```typescript
"Process all new orders: print labels, update tracking, and send to warehouse"

async function processDailyOrders() {
  // 1. Get unfulfilled orders
  const { orders } = await mcp.useTool('fulfillment_getOrders', {
    filter: 'orderfulfillmentstatus:{NOT_STARTED}',
    limit: 100
  });

  console.log(`ðŸ“¦ Processing ${orders.length} orders`);

  for (const order of orders) {
    // 2. Generate shipping label
    const label = await mcp.useTool('fulfillment_createShippingLabel', {
      order_id: order.orderId,
      shipping_service: 'USPS_PRIORITY',
      label_size: 'LABEL_4X6'
    });

    // 3. Print to warehouse printer
    await printLabel(label.labelUrl);

    // 4. Create packing slip
    const packingSlip = generatePackingSlip(order);
    await printPackingSlip(packingSlip);

    // 5. Mark as in progress
    console.log(`âœ… Order ${order.orderId} ready for packing`);
  }

  // 6. Export to warehouse management system
  await exportToWMS(orders);

  return `âœ… Processed ${orders.length} orders`;
}
```

<Tip>
Integrate with your warehouse management system (WMS) or use automation platforms like Zapier to trigger this workflow when new orders arrive.
</Tip>

### Use Case 2: Same-Day Shipping Guarantee

**Problem:** Committed to same-day shipping but manual processing takes too long.

**Solution:** Real-time order monitoring and instant label generation.

```typescript
"Monitor for new orders and auto-generate labels for immediate shipping"

async function autogenerateLabels() {
  const cutoffTime = '14:00'; // 2 PM cutoff for same-day shipping

  setInterval(async () => {
    const now = new Date();
    const currentTime = now.toTimeString().substring(0, 5);

    if (currentTime < cutoffTime) {
      // Get orders from last 5 minutes
      const fiveMinutesAgo = new Date(now.getTime() - 5 * 60000).toISOString();

      const { orders } = await mcp.useTool('fulfillment_getOrders', {
        filter: `creationdate:[${fiveMinutesAgo}..]`,
        limit: 50
      });

      for (const order of orders) {
        // Generate label immediately
        const label = await mcp.useTool('fulfillment_createShippingLabel', {
          order_id: order.orderId,
          shipping_service: determineBestService(order.shippingAddress)
        });

        // Send to fulfillment team
        await notifyWarehouse(order, label);
        console.log(`âš¡ Auto-generated label for ${order.orderId}`);
      }
    }
  }, 5 * 60000); // Check every 5 minutes
}
```

### Use Case 3: Return Automation

**Problem:** Manual return processing delays refunds and hurts customer satisfaction.

**Solution:** Automated return approval and refund processing.

```typescript
"Auto-approve returns within policy and issue refunds"

async function processReturns() {
  // Get pending return requests
  const returns = await mcp.useTool('fulfillment_getReturns', {
    states: ['RETURN_REQUESTED']
  });

  for (const returnRequest of returns.returns) {
    const order = await mcp.useTool('fulfillment_getOrder', {
      order_id: returnRequest.orderId
    });

    // Check if within return window (30 days)
    const orderDate = new Date(order.creationDate);
    const daysSinceOrder = (Date.now() - orderDate.getTime()) / (1000 * 60 * 60 * 24);

    if (daysSinceOrder <= 30) {
      // Auto-approve
      await mcp.useTool('fulfillment_processReturnRequest', {
        return_id: returnRequest.returnId,
        decision: 'ACCEPT',
        return_label_provided: true
      });

      // Generate return label
      const returnLabel = await mcp.useTool('fulfillment_createReturnShippingLabel', {
        return_id: returnRequest.returnId
      });

      // Notify buyer
      console.log(`âœ… Return approved for order ${order.orderId}`);
    } else {
      console.log(`âš ï¸ Return request outside window: ${order.orderId}`);
    }
  }
}
```

    **Always provide tracking information:**

    1. **Upload within 24 hours of shipping**
    2. **Use valid tracking numbers** - Test before submitting
    3. **Match carrier to tracking** - USPS tracking with USPS carrier
    4. **Update delivery status** - Mark delivered when confirmed

## Troubleshooting

<AccordionGroup>
  <Accordion title="'Invalid tracking number' error">
    **Cause:** Tracking number format doesn't match the carrier.

    **Solution:**
    - USPS: 20-22 digits starting with 94, 93, or 92
    - UPS: Starts with "1Z" followed by 16 characters
    - FedEx: 12-14 digits
    - Verify tracking on carrier website before submitting
  </Accordion>

  <Accordion title="'Order already fulfilled' error">
    **Cause:** Attempting to create a shipping fulfillment for an order that's already marked as shipped.

    **Solution:** Check order status first with `fulfillment_getOrder`. If you need to add another package, ensure you're using different line items.
  </Accordion>

  <Accordion title="Shipping label generation fails">
    **Cause:** Insufficient information or invalid address.

    **Solution:**
    - Verify buyer's shipping address is complete
    - Check package dimensions and weight are provided
    - Ensure shipping service is available for destination
    - Verify you have sufficient balance in eBay shipping account
  </Accordion>

  <Accordion title="Return not appearing in system">
    **Cause:** eBay's return processing may have a delay.

    **Solution:**
    - Wait 15-30 minutes and retry
    - Use `fulfillment_getReturns` with date filter
    - Check if return was initiated through eBay Money Back Guarantee (different workflow)
  </Accordion>
</AccordionGroup>
